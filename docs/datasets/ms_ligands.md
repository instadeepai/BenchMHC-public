# MS ligands

## Original Data

The MS ligands dataset used in BenchMHC originates from the [NetMHCpan-4.1 training and evaluation data](https://services.healthtech.dtu.dk/suppl/immunology/NAR_NetMHCpan_NetMHCIIpan/).

### Training

The data includes both binding affinity (BA) and eluted ligand (EL) data from HLA
and non-HLA alleles. The EL data is further categorized as single-allelic (SA) - where the MHC
allele associated to the ligand is known, and multi-allelic (MA) - where only the set of MHC alleles
present in the sample is known. This dataset has been split by the original paper into 5
cross-validation folds by minimizing the 8mers’ overlap across folds. These 5 folds were used to
create 5 train/tune splits (4 folds merged in train, the remaining fold used as tune).

### Evaluation

The HLA/SA MS ligands evaluation set from NetMHCpan-4.1[^1] covers a total of **45,416 MS MHC-I
eluted ligands each associated to one of 36 alleles**. As illustrated in the following figure, all
HLA alleles are included in the training dataset. As explained in the original paper, each set of
peptides associated to one allele was enriched, length-wise, with negative decoy peptides of 5 times
 the amount of ligands of the most abundant
peptide length. To ensure the independent test set’s orthogonality, NetMHCpan-4.1 removed from this
data set positive peptides that overlap with the training data.

## V1.0.0

The v1.0.0 dataset addresses a bias identified in
 the original data: **a human/non-human signal** in the data that models could learn instead of
learning true MHC-peptide binding patterns.

- **Processing impacting the hits**

In this version, unmatched hits are removed (peptides that could not be matched to any protein in the
reference proteomes).

- **Processing impacting the decoys**

All decoys are regenerated by sampling them within the same source protein as their corresponding hit.
The regeneration process is structured in two main steps, detailed in
[assign-protein-features](../reference/api/assign_protein_features.md) and
[generate-decoys](../reference/api/generate_decoys.md) sections.
This strategy eliminates the **human/non-human species bias** by ensuring decoys are drawn from
the same protein sources as hits, while maintaining specific properties (length, cysteine content).
Each decoy generated has been assigned the partition of its hit, and this partition was preserved
from the original NetMHCPan4.1 dataset.
The hit:decoy ratio is set to 1:99.

- **Processing impacting other columns**

The dataset is enriched with additional columns `protein_id` and `origin_species`.

??? note "Allele filtering script"
    The filtering was performed as follows:

    ```bash
    # Pull the MA training data
    mkdir -p data/netmhcpan-4.1/v1.3.0/training/
    gcloud storage cp -r gs://bench-mhc/data/v1.3.0/training/ma/ data/netmhcpan-4.1/v1.3.0/training/

    # Run the filtering script
    uv run allele_filtering.py
    ```

    File `allele_filtering.py`

    ```python
    """Remove samples with allele == 'amani.1' or 'gb1.7' from split CSV files."""

    import polars as pl
    import glob
    from pathlib import Path

    data_dir = Path("data/netmhcpan-4.1/v1.3.0/training/ma")
    csv_files = sorted(glob.glob(str(data_dir / "split_*.csv")))

    alleles_to_remove = {"amani.1", "gb1.7"}

    for csv_file in csv_files:
        lf_data = pl.scan_csv(csv_file)

        lf_filtered = lf_data.filter(~pl.col("allele").is_in(alleles_to_remove))

        lf_filtered.collect().write_csv(csv_file)
    ```
